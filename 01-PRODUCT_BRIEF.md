# Briefing do Produto — Cat Miner E-commerce

## Contexto

A Cat Miner vende ASICs para mineração de criptomoedas no Brasil. O site atual funciona como vitrine e apresenta equipamentos, preço, consumo, rentabilidade estimada, garantia e botão de compra via WhatsApp.

O novo projeto transforma essa vitrine em e-commerce simples, mantendo o atendimento humano forte, mas permitindo que o cliente compre diretamente com PIX.

## Posicionamento

A loja precisa comunicar:

- Tecnologia.
- Confiança.
- Equipamentos revisados.
- Suporte especializado.
- Compra simples.
- Pagamento seguro via PIX.
- Atendimento humano para produto de alto ticket.

## Público-alvo

- Pessoas que querem começar ou expandir operação de mineração.
- Compradores que já conhecem ASICs e buscam preço/estoque.
- Investidores que querem entender retorno estimado.
- Clientes que precisam de suporte para escolher equipamento.

## Promessa comercial permitida

A loja pode dizer:

- Equipamentos revisados.
- Pagamento via PIX.
- Garantia de 30 dias, se esse for o padrão real da operação.
- Suporte especializado.
- Entrega para todo Brasil, se a operação suportar.
- Simulação de rentabilidade.

## Promessa comercial proibida

Evitar:

- Rendimento garantido.
- ROI garantido.
- Lucro certo.
- Prazo garantido de retorno.
- Linguagem que faça parecer investimento sem risco.

## Copy base

Título principal:

> ASICs revisados para mineração de criptomoedas no Brasil

Subtítulo:

> Equipamentos testados, suporte especializado e pagamento seguro via Pix.

CTAs:

- Ver equipamentos
- Comprar com Pix
- Falar com especialista
- Tirar dúvida sobre este equipamento

## Diferenciais

- Equipamentos testados.
- Atendimento consultivo.
- Compra rápida via PIX.
- Produtos com dados técnicos claros.
- Garantia indicada na página.
- Aviso transparente sobre rentabilidade.

## Riscos que o produto precisa mitigar

1. Cliente achar que rentabilidade é garantida.
2. Cliente comprar equipamento incompatível com energia/local.
3. Cliente reclamar de equipamento usado por marcas de uso.
4. Cliente pagar e não receber confirmação.
5. Admin baixar estoque duas vezes por webhook duplicado.
6. Loja parecer improvisada e reduzir conversão.

## Estratégia do MVP

O MVP deve vender poucos produtos, com alta clareza técnica e fluxo simples. Não precisa de recursos avançados como cupons, login do cliente, avaliações, simulador completo ou frete automático complexo.

O foco é:

- Produto claro.
- Compra simples.
- PIX funcionando.
- Admin funcional.
- Estoque seguro.
- Layout premium.
